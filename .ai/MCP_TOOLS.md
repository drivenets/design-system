# DS MCP Tools Reference

The MCP server exposes tool schemas automatically via the protocol — connected
agents discover them on startup. This file documents the **recommended workflow**
for agents that aren't yet connected.

## Server

- **Endpoint**: `http://localhost:3000/mcp`
- **Setup**: See `tools/mcp-server/README.md`
- **Architecture**: See `tools/mcp-server/ARCHITECTURE.md`

## Workflow

```
1. list-all-documentation(withStoryIds: true)  → discover component/docs IDs
2. get-documentation(id: "components-ds-button")  → full props, stories, code snippets
3. get-documentation-for-story(componentId, storyName)  → specific story variant
```

Always start with step 1 to get valid IDs. Do not guess component IDs — they
are generated by Storybook and may not match the component name exactly.
